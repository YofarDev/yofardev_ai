# Localization (L10n)

This directory contains the internationalization (i18n) and localization (l10n)
configuration for the application.

## Files

- `app_localizations_en.arb` - English translations (template file)
- `app_localizations_fr.arb` - French translations

## Configuration

The localization setup is configured via `l10n.yaml` in the project root:

```yaml
arb-dir: lib/core/l10n
template-arb-file: app_localizations_en.arb
output-localization-file: app_localizations.dart
output-dir: lib/core/l10n/generated
nullable-getter: false
```

**Note:** The `synthetic-package` option has been deprecated and removed.
We now use `output-dir` to specify where generated files are created.
The generated directory `lib/core/l10n/generated` is excluded from version control
as the files are regenerated automatically.

## Important Notes

**DO NOT manually edit the generated `app_localizations.dart` file.**

The `AppLocalizations` class is automatically generated by Flutter's codegen
from the ARB files in this directory. It will be regenerated each time you run:

```bash
flutter gen-l10n
```

Or when you build/run the app (if `generate: true` is set in pubspec.yaml).

The generated file is located at:
`lib/core/l10n/generated/app_localizations.dart`

## How to Use

### 1. Adding New Translations

1. Add your new translation key to the template `.arb` file (English) with metadata:

```json
{
  "myNewKey": "Hello World",
  "@myNewKey": {
    "description": "A friendly greeting message"
  }
}
```

2. Add the same key to all other locale files (without metadata):

```json
{
  "myNewKey": "Bonjour le monde"
}
```

3. Run the following command to generate the localization code:

```bash
flutter gen-l10n
```

Or simply run your app - the code will be generated automatically if
`generate: true` is set in `pubspec.yaml`.

### 2. Using Translations in Code

Import the generated localization class and use it in your widgets:

```dart
import 'package:flutter/material.dart';
import 'core/l10n/generated/app_localizations.dart';

class MyWidget extends StatelessWidget {
  const MyWidget({super.key});

  @override
  Widget build(BuildContext context) {
    final l10n = AppLocalizations.of(context);
    return Text(l10n.myNewKey);
  }
}
```

### 3. Adding New Languages

1. Create a new `.arb` file with the appropriate locale code in this directory:
   - `app_localizations_es.arb` for Spanish
   - `app_localizations_de.arb` for German
   - etc.

2. Copy all keys from the template file and translate the values.

3. Add the locale to your app's `supportedLocales` in `main.dart`:

```dart
supportedLocales: const <Locale>[
  Locale('en'), // English
  Locale('fr'), // French
  Locale('es'), // Spanish (new)
],
```

4. Run `flutter gen-l10n` to regenerate the localization files.

### 4. Supported Locales

The app currently supports:
- English (`en`) - template locale
- French (`fr`)

## Best Practices

1. **Use nested keys for organization**: Group related translations
   under categories like `home`, `common`, `counter`, etc.

2. **Keep keys descriptive**: Use clear, descriptive key names that
   indicate their purpose.

3. **Add metadata for translators**: Include `@key` metadata in the template
   file with descriptions to help translators understand context.

4. **Maintain consistency**: Keep the same key structure across all
   locale files.

5. **Generate after changes**: Always run `flutter gen-l10n` after
   modifying `.arb` files to see changes in your app.
